After opening the terminal, there a few commands you can use:

  • pwd - prints in a /slash/ format where you're located currently.
  • ls - prints a list of the items contained within the current directory.
  • -a - added to ls>, shows all hidden items, including the previous directory.
  • -F - added to ls, adds the type of item to the end of the name (eg directory/).
  • . - signifies the current location (useful somehow later).
  • .. - signifies the previous (vertically, not chronologically) directory.
  • cd - changes your location to the specified item (eg not looking, moving).
  • / - when at the beginning of a path or alone, signifies the root directory (the origin of all items).
  • ~ - signifies the current user's home directory (eg /Users/ailsa).

Also, to clear up, a couple definitions:

  • Absolute Path - the path from the root directory to an item.
  • Relative Path - a path used to get from the current location to an item (eg if the location is the root directory, the relative path is the same as the absolute path).

Now, this *seems* like kind of useless info (at least I thought so). Why would you learn all this crap if you can just go to the Finder and see it all, complete with little folder icons for "directories" and previews for documents, applications, files, etc?

Turns out there's a couple reasons. First, there's stuff you can see from the terminal that you can't from the Finder, and it's laid out more clearly (in a sense, at least, and that sense is not visual). Second, getting used to the terminal is vital for later crap, especially if you want to run some powerful program for parsing data or something. I have a feeling I'm gonna be doing that pretty soon based on the "exercise" files the site had me download before beginning.

The end of the lesson has an interesting tidbit about command syntax. It has an example command that looks like this:

  • $ ls -F /

The $, or on my Mac the %, is the prompt. In my terminal it has some crap that comes before it. the "ls" is the actual command, like the verb of the sentence. This one is telling the terminal to list something in alphabetical order, etc. Third is the option (or "switch" or "flag"), the "-F", which changes the behavior of the command, and last is the argument, which is like the direct object - the "/", which is shorthand for the root directory. So all in all, the line is telling the computer to list all the items stored one layer deep in the root directory while marking the type of item at the end of the block.

But back to the content! Learned a couple new, useful things:

  • mkdir - creates a new directory within the current location.
  • nano - preceding a .txt name, creates a text file and opens it in nano.
  • touch - preceding a .txt name, creates an empty text file.
  • -l - an option for listing the "stats" of a file.
  • rm - removes a file
  • mvmoves a file, can be used to "rename" a file by moving it to the same location under a new name (eg current_directory/new_name)
  • cp - does the same thing as **mv**, except leaves behind a copy. Putting /directory at the end of multiple items copies them to that directory.
  • -r - an option ("recursive") to copy a directory and all of its contents (can only copy or remove a directory with this option).
  • * - a wildcard that represents zero or more characters
  • ? - a wildcard that represents exactly one character
  • -p - when applied to a mkdir command, creates any intermediate directories in a path (eg creates 2016, data, and processed nested in order when given 2016/data/processed)
  • code . - opens the current location in VSCode.
