After that, it's just a matter of making changes, staging, and committing.

Some new and exciting git commands to log:

  • git status - shows the status of modified, staged, and committed on the files that have been messed with recently. Shows how many commits you are ahead of your last sync with GitHub.
  • git add - adds a file to the staging area, which will be committed all at once when given the command.
  • git commit -m - commits files; if followed by quotes adds a description of the changes made (now that I've altered the settings, I can omit the -m in VSCode and just type in the message when given the prompt.)
  • git log - shows the commits you've made recently, as well as who made them, the time, and their descriptions.
  • git push - syncs up the commits with GitHub, the origin of the repo.

Finally, the last thing to do is sync all this with GitHub, and it'll show up! Just don't make changes to anything from inside GitHub, and use the "best practices" such as atomic commits, which only change one feature or task in order to keep problems easy to spot and make it easier for other people going through the commits.
